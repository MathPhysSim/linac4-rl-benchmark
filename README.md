# Linac4 RL Benchmark

Reinforcement Learning benchmark for autonomous beam steering on the **CERN Linac4** transfer line, using measured response matrices from October 2019.

>
> S. Hirlaender and N. Bruchon, *"Sample-efficient reinforcement learning for CERN accelerator control,"*

> [DOI: 10.1103/PhysRevAccelBeams.23.124801](https://doi.org/10.1103/PhysRevAccelBeams.23.124801)

## Scientific Context

**Linac4** is a 160 MeV H-minus linear accelerator at CERN that feeds the Proton Synchrotron Booster (PSB). The beam travels through a transfer line equipped with:

- **Corrector magnets** (16 horizontal + 16 vertical): actuators that steer the beam.
- **Beam-position monitors (BPMs)** (16 horizontal + 16 vertical): sensors that measure the transverse orbit.

The relation between corrector settings **u** and beam positions **x** is approximately linear and described by the **response matrix R**:

```
x = R · u
```

This repository contains **R** as measured at low intensity on 21--22 October 2019, together with a Gym simulation environment that adds small non-linear perturbations, and several RL/optimisation algorithms benchmarked on the steering task.

The objective is to find corrector settings that bring the beam positions as close as possible to a reference orbit, minimising the root-mean-square (RMS) deviation.

## Environment

### `simple_environment_linac4.py` -- `simpleEnv`

A custom OpenAI Gym environment (`gym.Env`) that simulates the Linac4 beam-steering problem.

| Property | Value |
|---|---|
| **Action space** | `Box(-1, 1, shape=(16,))` -- normalised corrector settings |
| **Observation space** | `Box(-1, 1, shape=(16,))` -- normalised BPM readings |
| **Reward** | Negative RMS deviation from the reference trajectory |
| **Episode termination** | Reward < -10, reward > -0.25, or step > 25 |

**Dynamics:**

```
state = (R + noise) · action + action^T · N · action
```

where `R` is the measured response matrix, `noise` is i.i.d. Gaussian (scale 0.01), and `N` is a fixed non-linear perturbation matrix (also scale 0.01).

The reference trajectory is a vector of ones. The initial state is generated by applying a large random corrector setting (`5 * randn`) to simulate a displaced orbit.

## Algorithms

### 1. NAF -- Normalized Advantage Function

Based on [Gu et al., 2016](http://arxiv.org/abs/1603.00748). NAF extends DQN to continuous action spaces by decomposing the Q-function as:

```
Q(s, a) = V(s) + A(s, a)
```

where the advantage `A` is parameterised as a quadratic form, enabling closed-form maximisation over actions.

Two implementations are provided:

| Variant | Script | Library | Key Feature |
|---|---|---|---|
| **Original NAF** | `kims_naf.py` | `NAF_old/` | Separate pred/target networks, no PER |
| **New NAF (PER)** | `new_naf.py` | `NAF_new/` | Prioritized Experience Replay, SpinningUp-style MLP |

### 2. DDPG -- Deep Deterministic Policy Gradient

Script: `spinning_up_script.py`

Uses the OpenAI SpinningUp DDPG implementation (`spinup.ddpg`) as a baseline comparison.

### 3. Classical Optimisation Baselines

Script: `plain_optimization.py`

Non-RL baselines using derivative-free optimisation:

| Algorithm | Library | Notes |
|---|---|---|
| **COBYLA** | `scipy.optimize` | Constrained optimisation by linear approximation |
| **Powell** | `scipy.optimize` | Direction-set method |
| **Bayesian Optimisation** | `bayesian-optimization` | Gaussian-process-based global optimisation |

## Hyperparameters

Default hyperparameters used across the NAF scripts:

| Parameter | Value | Description |
|---|---|---|
| `discount` | 0.999 | Discount factor (gamma) |
| `batch_size` | 10 | Minibatch size for replay |
| `learning_rate` | 1e-3 | Adam optimiser learning rate |
| `max_steps` | 500 | Maximum steps per episode |
| `update_repeat` | 7 | Q-learning updates per environment step |
| `max_episodes` | 100--700 | Total training episodes (varies by script) |
| `tau` | 0.001 | Soft target-network update rate |
| `hidden_dims` | [16, 16] | Hidden layer sizes |
| `activation` | tanh | Hidden layer activation function |
| `noise_scale` | 1/(episode+1) | Exploration noise (decaying) |

## Data

The `DATA/` directory (gitignored) contains ~100 HDF5 files recorded from the Linac4 control system on **21--22 October 2019**. Each file is a snapshot of corrector settings and BPM readings.

| File | Description |
|---|---|
| `linac4Communicator_*.h5` | Raw measurement snapshots |
| `init_states.h5` | Initial beam states |
| `data_all` | Aggregated dataset (generated by `read_data.py`) |

### Data Processing

- **`DATA/read_data.py`**: Aggregates all HDF5 files into a single pandas DataFrame (`data_all`), filtering out BCT columns.
- **`DATA/generate_model.py`**: Trains a Keras neural network surrogate model (400-300-2 architecture) on the aggregated data with early stopping.

---

## Response Matrix

The response matrix `linac4_rm.pcl` is a pickled 32x32 pandas DataFrame containing the measured coupling between all 32 correctors (16 horizontal + 16 vertical) and all 32 BPMs.

- **Horizontal plane**: rows 0--15, columns 0--15
- **Vertical plane**: rows 17+, columns 16+

The heatmap visualisations (`Response_matrix_hor.png`, `Response_matrix_ver.png`) can be regenerated with:

```bash
python plot_response.py
```

## Usage

### Prerequisites

This project was developed with **Python 3.6** and **TensorFlow 1.x** (circa 2019). TensorFlow 1.x is no longer installable on modern systems, so the code is preserved as a historical reference.

**Historical dependencies:**

```
tensorflow>=1.12,<2.0
gym>=0.10,<0.26
numpy
pandas
matplotlib
seaborn
scipy
bayesian-optimization
h5py
```

### Running the Scripts

Each script is self-contained and can be run directly:

```bash
# Train the original NAF agent (700 episodes)
python kims_naf.py

# Train the new NAF agent with PER (100 episodes)
python new_naf.py

# Verify a pre-trained NAF checkpoint
python kims_naf_verification.py

# Train DDPG via SpinningUp
python spinning_up_script.py

# Run COBYLA optimisation baseline
python plain_optimization.py

# Visualise the response matrix
python plot_response.py

# Plot initial beam states from measurement data
python reference_orbit.py
```

## Bundled Libraries

### `NAF_old/` and `NAF_new/`

Forked from [carpedm20/naf-tensorflow](https://github.com/carpedm20/naf-tensorflow). The `NAF_new` variant adds:
- A proper `ReplayBuffer` (FIFO, fixed-size NumPy arrays).
- `ReplayBufferPER` for Prioritized Experience Replay.
- SpinningUp-style MLP construction (`core.mlp_normalized_advantage_function`).

### `spinup/`

A subset of [OpenAI SpinningUp](https://github.com/openai/spinningup) bundled for convenience. Provides DDPG, TD3, SAC, and PPO implementations.
